Axum is the most popular Rust web framework. Ultimo builds on the same Hyper + Tokio foundation but takes a different approach to developer experience. Here's an honest comparison to help you decide.

## The Short Version

| Aspect | Ultimo | Axum |
|--------|--------|------|
| **Core** | Hyper 1.0 + Tokio | Hyper 1.0 + Tokio |
| **TypeScript client** | Auto-generated | Manual / third-party |
| **JSON-RPC** | Built-in | Not included |
| **OpenAPI** | Auto-generated | Via `utoipa` or similar |
| **WebSocket** | Built-in pub/sub | Via `axum::extract::ws` |
| **Routing** | O(1) hash lookup | Trie-based |
| **Maturity** | Pre-1.0 (0.5.x) | Stable (0.8.x) |
| **Ecosystem** | Growing | Large |

## Performance

Both frameworks sit on Hyper 1.0, so raw throughput is comparable. Ultimo's O(1) static route lookup gives a slight edge in apps with many routes, while Axum's trie-based router handles complex path patterns more flexibly.

```
# Benchmark: 1000 routes, simple JSON response
# (Criterion, i7-13700K, Linux 6.x)

Ultimo:  158,432 req/s  (p99: 0.8ms)
Axum:    155,891 req/s  (p99: 0.9ms)
```

The difference is negligible. Choose based on features, not benchmarks.

## Developer Experience: Hello World

**Ultimo:**

```rust
use ultimo::prelude::*;

async fn hello(ctx: Context) -> Result<impl Into<Response>> {
    ctx.text("Hello, World!")
}

#[tokio::main]
async fn main() -> Result<()> {
    Ultimo::new()
        .get("/", hello)
        .port(3000)
        .run()
        .await
}
```

**Axum:**

```rust
use axum::{routing::get, Router};

async fn hello() -> &'static str {
    "Hello, World!"
}

#[tokio::main]
async fn main() {
    let app = Router::new().route("/", get(hello));
    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000")
        .await
        .unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

Both are concise. Axum uses extractors as function parameters; Ultimo passes a `Context` object. Axum's approach is more type-level; Ultimo's is more runtime-flexible.

## TypeScript Client Generation

This is Ultimo's killer feature. Define your types in Rust, and the CLI generates a complete TypeScript client:

```rust
#[derive(Serialize, TS)]
struct User { id: u64, name: String }

rpc.query("getUser", get_user);
// → TypeScript client with full types, zero manual work
```

With Axum, you'd need to manually write TypeScript types, use a code generator like `ts-rs` separately, or maintain an OpenAPI spec and generate from that. Ultimo eliminates this entire workflow.

## JSON-RPC Support

Ultimo has first-class JSON-RPC alongside REST in the same app:

```rust
let mut rpc = RpcRegistry::new();
rpc.query("getUser", get_user);
rpc.mutation("createUser", create_user);

Ultimo::new()
    .get("/api/users", list_users)   // REST
    .rpc("/rpc", rpc)                // JSON-RPC
    .run().await
```

Axum has no built-in JSON-RPC. You'd use `jsonrpsee` or implement it manually.

## When to Choose Ultimo

- You're building a **full-stack app** with a TypeScript frontend
- You want **automatic client generation** from Rust types
- You need both REST and JSON-RPC in one service
- You value **convention over configuration**
- You want built-in WebSocket pub/sub, sessions, CSRF, and auth

## When to Choose Axum

- You need the **largest ecosystem** of middleware and extractors
- You're building **infrastructure** (proxy, gateway) rather than application APIs
- You need **maximum flexibility** in how extractors compose
- You want the safety of a **stable, post-1.0 API**
- Your team already knows Axum well

## Can You Use Both?

Yes. Ultimo and Axum share the same Hyper + Tokio runtime. You can run them as separate services in the same cluster, or gradually migrate routes. Ultimo's handler signature is different from Axum's, but the mental model (async function → response) is the same.

## Conclusion

Axum is the safe, mature choice with a massive ecosystem. Ultimo is the opinionated choice that eliminates boilerplate — especially the TypeScript boundary. If you're tired of keeping your frontend types in sync with your Rust backend, Ultimo exists to solve exactly that problem.
